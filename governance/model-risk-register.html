Risk Management LLM Toolkit — Governance-of-Governance

Model Risk Register

Every agent in this toolkit contains AI components making judgment calls — generating narratives, classifying updates, recognizing triggers. This register treats those components as models under formal inventory and governance, modeled on SR 11-7 (the Fed/OCC's Model Risk Management guidance): inventory, risk tiering, validation status, and a documented remediation path for gaps.

● No API calls — this register is static governance content, always available regardless of AI status

Governance cadence

  • Model inventory review: any new agent or AI-powered feature gets an entry here before being marked "built" in the main README's status table — not after.
  • Re-validation trigger: any model version pin change requires re-running every High and Medium tier model's eval suite before the change is treated as adopted.
  • Sign-off: Sadiq is the sole Model Risk Owner for all entries in this sandbox. In production, High-tier models would require sign-off independent of the model's builder.
